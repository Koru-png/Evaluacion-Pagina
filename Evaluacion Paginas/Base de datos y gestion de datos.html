¿Qué es una Base de Datos?

Una base de datos es un almacén organizado de información que permite almacenar, consultar, actualizar y gestionar datos de forma confiable. Un Sistema de Gestión de Bases de Datos (DBMS) abstrae la complejidad de acceso, garantiza integridad, concurrencia y recuperación ante fallos.

Modelos de Datos

  • Relacional: Datos en tablas (filas/columnas) usando claves primarias y foráneas (SQL).
  • NoSQL: Datos no estructurados o semi-estructurados:
    • Documentos (MongoDB)
    • Clave-Valor (Redis)
    • Grafos (Neo4j)
    • Columnas Anchas (Cassandra)
  • Orientado a Grafos: Entidades y relaciones como nodos/aristas, ideal para redes sociales.

SQL vs NoSQL

SQL

  • Estructura: Tablas con esquema fijo.
  • ACID: Transacciones garantizadas.
  • Ejemplos: MySQL, PostgreSQL, Oracle, SQL Server.

NoSQL

  • Esquema flexible o sin esquema.
  • Escalabilidad horizontal.
  • Ejemplos: MongoDB, Cassandra, Couchbase, Redis.

Normalización e Integridad

La normalización minimiza redundancias y asegura consistencia. Formas normales comunes:

  • 1FN: Eliminar grupos repetidos.
  • 2FN: Eliminar dependencias parciales.
  • 3FN: Eliminar dependencias transitivas.
  • BCNF: Versión más estricta de 3FN.

Integridad garantizada con claves primarias, foráneas y restricciones (UNIQUE, NOT NULL, CHECK).

Índices y Rendimiento

Los índices aceleran consultas al costo de espacio adicional y mayor latencia en inserciones/actualizaciones. Tipos principales:

  • B-Tree: Óptimo para búsquedas de rango.
  • Hash: Rápido para búsquedas por igualdad.
  • Texto Completo: Búsqueda por palabras en campos textuales.
  • Compuestos: Índices sobre varias columnas.

Transacciones y Concurrencia

Una transacción es una unidad atómica de trabajo que debe cumplir ACID:

  • Atomicidad: Todo o nada.
  • Consistencia: Mantiene reglas de integridad.
  • Aislamiento: Evita interferencias entre transacciones.
  • Durabilidad: Los cambios persisten tras fallos.

Niveles de aislamiento: READ UNCOMMITTED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE (controlan bloqueos y concurrencia).

Backup y Recuperación

Protege los datos ante fallos con estas estrategias:

  • Backup Completo: Copia de toda la base de datos.
  • Backup Diferencial: Cambios desde el último backup completo.
  • Backup Incremental: Cambios desde el último backup (completo o incremental).
  • Replicación: Réplica en tiempo real a servidores secundarios para alta disponibilidad.

Validar periódicamente los backups garantiza su fiabilidad cuando sean necesarios.

Seguridad en Bases de Datos

Para proteger la información sensible:

  • Cifrado en Tránsito: TLS/SSL para conexiones seguras.
  • Cifrado en Reposo: AES para datos almacenados.
  • Control de Acceso: Roles, privilegios mínimos, autenticación multifactor.
  • Auditoría: Monitoreo de accesos y actividad sospechosa.
  • Actualizaciones: Parches regulares para cerrar vulnerabilidades.

Un diseño seguro previene inyecciones SQL, escalada de privilegios y accesos no autorizados.

Big Data y Analytics

Para procesar grandes volúmenes de datos empleamos:

  • Hadoop: HDFS, MapReduce, Hive.
  • Spark: Procesamiento en memoria para análisis rápido.
  • Data Warehouses: Snowflake, Redshift, BigQuery.
  • Data Lakes: Repositorios en bruto (AWS S3, Azure Data Lake).

ETL/ELT transforma datos para alimentar modelos predictivos, dashboards y BI.
